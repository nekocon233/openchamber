# 데스크톱 터널

데스크톱 앱은 **Settings → OpenChamber → Tunnel**에서 공개 터널을 만들 수 있습니다. 이 경로에서는 OpenChamber를 CLI로 시작할 필요가 없습니다.

## 공급자 설치

OpenChamber는 사용자의 컴퓨터에서 공급자 CLI를 실행합니다. Cloudflare 또는 Ngrok을 사용한다면 먼저 원하는 공급자를 설치하세요:

```bash
brew install cloudflared
brew install ngrok
```

Cloudflare는 `cloudflared`를 사용합니다. Ngrok은 ngrok 계정과 ngrok 대시보드의 authtoken이 필요합니다:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

FRPC는 직접 설치할 필요가 없습니다. OpenChamber가 FRPC 0.70.0을 자동으로 다운로드하고 관리합니다.

## 앱에서 시작

1. **Settings → OpenChamber → Tunnel**을 엽니다.
2. **Cloudflare**, **Ngrok** 또는 **FRPC**를 선택합니다.
3. Cloudflare 또는 Ngrok에서는 빠른 터널을 시작하고, FRPC에서는 설정값을 입력합니다.
4. 휴대폰에서 생성된 QR 코드를 스캔합니다.

Ngrok은 현재 빠른 터널을 지원합니다. Cloudflare는 빠른 터널과 Cloudflare 관리형 모드를 지원합니다. FRPC는 자체 FRPS 서버를 통한 `managed-remote` TCP 및 HTTP 가상 호스트 엔드포인트를 지원합니다.

## FRPC 구성

FRPC 설정에서는 프록시 유형으로 **TCP**와 **HTTP 가상 호스트**를 선택할 수 있습니다. FRPS 주소, 제어 포트, 신뢰할 CA 인증서 파일, 공유 토큰을 비공개 토큰 필드에 입력하세요. 두 유형 모두 OpenChamber가 로컬 포트를 자동으로 제공합니다. FRPS는 SAN이 FRPS 주소와 일치하는 인증서를 제공해야 하며, OpenChamber는 이 신원을 검증하고 인증서 검증을 비활성화하지 않습니다.

- **TCP:** 원격 매핑 포트와 정확한 외부 공개 HTTPS 오리진을 입력합니다. FRPC는 매핑된 포트에서 암호화되지 않은 HTTP를 전달하므로 해당 FRPS 포트는 루프백에서만 수신하고 선언한 URL에 Caddy 또는 다른 TLS 종단점을 배치하세요. OpenChamber는 HTTP, 잘못된 URL 또는 오리진 외 요소가 포함된 URL을 거부하며 FRPS 제어 주소에서 브라우저 TLS를 추정하지 않습니다.
- **HTTP 가상 호스트:** FRPS 라우팅 호스트(custom domain)와 외부 공개 hostname을 입력합니다. 공개 URL은 `https://<hostname>`입니다. Caddy는 포트 `443`에서 HTTPS를 종료하고 FRPS HTTP 가상 호스트 포트로 전달하며, `Host`는 custom domain으로, `X-Forwarded-Host`는 공개 hostname으로 설정합니다.

공유 FRPS 토큰은 FRPC 클라이언트를 인증하지만 HTTP 가상 호스트 사이의 격리를 제공하지는 않습니다.

## 접근 보호

공급자 URL 자체가 공개되어 있어도 OpenChamber는 자체 connect token으로 접근을 보호합니다. 생성된 연결 링크에는 일회용 token이 들어 있고 TTL이 있으며, 새 링크를 만들거나 터널을 중지/재시작하면 이전의 사용되지 않은 링크는 무효화됩니다.
